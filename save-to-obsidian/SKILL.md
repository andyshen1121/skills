# 保存内容到 Obsidian Vault

## Vault 路径发现

不要硬编码 vault 路径。每次执行时动态查找：

```bash
find ~/Documents -maxdepth 3 -name ".obsidian" -type d 2>/dev/null
```

- 找到 1 个：直接使用其父目录作为 vault 路径
- 找到多个：列出所有 vault 路径，询问用户选择存入哪个
- 找到 0 个：扩大搜索范围到 `~`（maxdepth 4），仍未找到则询问用户提供路径

## 目录规则

根据内容类型自动选择目录：

| 内容类型 | 目标目录 |
|----------|----------|
| 美股/投资/财务分析 | 根目录（暂无子目录，直接放根目录） |
| 技术方案/架构文档 | `Code/` |
| SQL 查询记录 | `Code/` |
| 日常笔记/记录 | 根目录 |
| 其他 | 根目录 |

如果目标目录不存在，先询问用户是否创建，不要自动创建。

## 执行步骤

### 1. 推断文件名
格式：`主题关键词_YYYYMMDD.md`

示例：
- `RKLB投资分析_20260302.md`
- `OSS账单分析_20260302.md`
- `托管商家SQL设计_20260302.md`

向用户确认文件名，允许修改。

### 2. 生成前置元数据
```markdown
---
date: YYYY-MM-DD
tags: [analysis]
source: claude-code
---
```

tags 根据内容类型补充，如 `[analysis, stock, investment]`、`[sql, backend]`。

### 3. 组合完整内容
前置元数据 + 原始分析内容（不修改内容本身）

### 4. 写入前确认
输出完整预览（含路径、文件名、元数据），等用户确认后再写入。
格式：
```
将写入：<vault路径>/[目录/]文件名.md
确认写入？(y/n)
```

## 规则
- 禁止不经确认自动写入文件
- 不修改用户提供的分析内容本身，只添加元数据头部
- 如文件已存在，明确告知并询问是覆盖还是追加
